## Assumptions

1. **"All features" = actionable features.** Browsing the sessions list is free. The paywall gates: start a workout, create, edit, and duplicate sessions.
2. **Trial clock starts on first app launch.** The start date is persisted to `trial_v1.json` via `expo-file-system` (same pattern as settings/sessions). It is never reset unless explicitly triggered in dev mode.
3. **Mock `purchasePremium()` stays as-is.** It sets `_isPremium = true` in memory. This resets on app restart â€” acceptable for a mock.
4. **The 5-session cap is removed.** `sessionLimit.ts` is deleted. During the trial, session creation is unlimited.
5. **`PaywallModal` replaces `Alert.alert('Premium Required', ...).`** It shows "Your 30-day free trial has ended" with a Purchase button.

## Open Question

The current `_isPremium` flag in `purchases.ts` is **not persisted** â€” it resets on every app restart. Paid status therefore does not survive a cold launch under the mock. Is this acceptable for now, or should purchase state also be persisted?
